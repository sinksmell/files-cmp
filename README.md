## 📖 项目简介

**Files-CMP** 是一个高性能的分布式文件对比工具，专为解决大规模文件对比场景而设计。当两台机器各自存储千万级小文件，且仅有少量文件存在差异时，本工具能够在低带宽、高延迟的网络环境下，以最小的内存占用和最快速度找出差异文件。

### 适用场景

- 🖥️ 两台机器各有大量文件（如 1000 万+）
- 📊 仅有极少数文件存在差异（如 3-5 个）
- 🌐 网络条件较差（如几百 MB 需要传输 10 小时）
- 💾 磁盘性能良好（SSD）
- 🔄 支持增量同步

---

## 🎯 核心特性

- ✅ **高效分组算法**: 采用平方根分组策略，大幅减少网络传输量
- ✅ **MD5 哈希校验**: 快速定位差异文件，避免全量传输
- ✅ **低内存占用**: 流式处理，适合大规模文件对比
- ✅ **智能对比**: 
  - 文本文件：逐行分析，高亮显示差异
  - 二进制文件：对比前 512 字节
- ✅ **增量同步**: 支持检测并同步增量变动
- ✅ **彩色输出**: 终端彩色高亮显示对比结果

## 🏗️ 架构设计

### 对比策略

```
┌─────────────┐                    ┌─────────────┐
│   Client    │ ◄──── HTTP ────►   │   Server    │
│  (客户端)    │                    │  (服务端)    │
└─────────────┘                    └─────────────┘
      │                                   │
      ▼                                   ▼
 1. 获取文件列表                      1. 初始化文件列表
 2. 计算文件 MD5                      2. 文件分组
 3. 按组发送请求                      3. 等待对比请求
 4. 接收差异文件列表                  4. 返回差异文件
 5. 发送差异文件                      5. 详细对比
```

### 工作流程

#### 第一阶段：分组对比

1. **文件列表获取**: 使用 `ls` 命令获取文件名列表（自动按字母序排列）
2. **智能分组**: 分组大小 = √文件总数（向下取整）
3. **计算组哈希**: 遍历文件集合，计算每个文件的 MD5 值，以 `文件名:MD5` 形式存入分组文件
4. **网络请求**: Client 向 Server 发送分组文件名及其 MD5 值
   - ✅ **相同**: 继续处理下一分组
   - ❌ **不同**: Server 定位差异文件，通知 Client 发送该文件

#### 第二阶段：精确定位

从 MD5 值不同的分组文件中找出具体差异文件：

1. Server 读取对应分组文件，建立 `Map[文件名]→MD5` 索引
2. 逐行比对 Client 传来的分组文件
3. 返回差异文件名集合给 Client

#### 第三阶段：文件内容对比

- 📝 **文本文件**: 分析独特内容，高亮显示变化
- 🖼️ **二进制文件** (JPG, MP4 等): 对比前 512 字节

## 🚀 快速开始

### 环境要求

- Go 1.x 或更高版本
- Linux / macOS / Windows

### 安装依赖

```bash
go get github.com/astaxie/beego
```

### 启动服务端

```bash
go run main.go
```

服务端将：
- 初始化 `./file/` 目录下的文件列表
- 自动进行文件分组，生成 `./group/` 目录下的分组文件
- 启动 HTTP 服务（默认端口 8080）

### 启动客户端

```bash
cd client
go run client.go
```

客户端将：
- 自动对比 `./file/` 文件夹下的所有文件
- 与服务端进行分组哈希对比
- 找出并上传差异文件进行详细对比

## 🔧 配置说明

### 核心常量 (`models/comm.go`)

| 常量 | 默认值 | 说明 |
|------|--------|------|
| `FILE_PATH` | `./file/` | 小文件存放路径 |
| `GROUP_PATH` | `./group/` | 分组文件保存路径 |
| `CMP_SIZE` | `512` | 非文本文件比较字节数 |

### 响应码

| 代码 | 含义 |
|------|------|
| `0000` | 文件 MD5 相同 |
| `0001` | 文件处理成功 |
| `1000` | 文件 MD5 不同 |
| `2000` | 请求错误 |
| `3001` | 文件保存错误 |
| `3002` | 文件对比错误 |

## 💡 性能优势

| 传统方法 | Files-CMP |
|---------|-----------|
| 全量传输所有文件 | 仅传输差异文件 |
| 内存占用高 | 流式处理，低内存 |
| 网络开销大 | 分组哈希，最小化传输 |
| 速度慢 | 并行处理，快速定位 |

**示例**: 1000 万文件，仅 5 个差异
- 传统方法：需传输全部文件（可能数百 GB）
- Files-CMP：仅传输分组哈希 + 5 个差异文件（几 MB 到几十 MB）
